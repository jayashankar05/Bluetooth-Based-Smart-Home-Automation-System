# Images – Placeholder Guide

This folder contains project images for the GitHub repository.

## Required Images

| Filename          | Description                                                              | Status      |
|-------------------|--------------------------------------------------------------------------|-------------|
| `setup.jpg`       | Complete hardware setup photo – all components connected on breadboard   | Add yours   |
| `working.jpg`     | Live working photo – relay LEDs ON with appliance switched ON            | Add yours   |
| `components.jpg`  | All individual components laid out neatly on a flat surface              | Add yours   |
| `output_demo.jpg` | Screenshot of the Bluetooth terminal app showing commands and responses  | Add yours   |

## Photography Tips

1. **Lighting:** Use natural light or a white LED panel. Avoid harsh shadows.
2. **Background:** Use a plain white or dark surface for component photos.
3. **Focus:** Ensure module labels (HC-05, Relay IN1–IN4, Arduino pins) are readable.
4. **Angle:** Slight overhead angle (45°) works best for circuit board photos.
5. **Resolution:** Minimum 1280×720 pixels. 1920×1080 recommended.
6. **Phone Camera:** Modern smartphone cameras are sufficient. Use Pro mode for better control.

## File Naming Convention

- Use lowercase with underscores: `setup.jpg`, `working.jpg`
- JPEG format preferred for photographs
- PNG format preferred for screenshots
- Keep file size under 5 MB per image for fast GitHub loading

## Adding Images to README

Once images are captured and placed in this folder, they are referenced in README.md using:

```markdown
![Setup Photo](images/setup.jpg)
![Working Demo](images/working.jpg)
```
